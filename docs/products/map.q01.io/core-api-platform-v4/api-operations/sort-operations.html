Sort Operations

Overview​

Sort operations enable drag-and-drop reordering of records by updating sequence field values. When a user moves an item from one position to another, CoreWrite automatically recalculates sequence numbers for affected records in a single transaction.

Key Characteristics:

  • Drag-and-drop reordering
  • Automatic sequence recalculation
  • Transaction-safe (ACID)
  • Supports conditional sorting (e.g., sort within category)
  • Updates LOWNER and LDATA audit fields
  • Maintains TREC state

Endpoint​

Method: POST Path: /api/v4/core/{DIM}/sort

Authentication: Required (Bearer token)

Permission: Grant ≥ 2 (same as UPDATE operations)

How Sort Works​

Conceptual Model​

Imagine a sortable list of products:

Initial State:
┌───────────────────────┐
│ A - seq: 1           │
│ B - seq: 2  ←active  │
│ C - seq: 3           │
│ D - seq: 4           │
│ E - seq: 5  ←over    │
│ F - seq: 6           │
└───────────────────────┘

User drags B to E's position (moving down):

Step 1: Shift items up
┌───────────────────────┐
│ A - seq: 1           │
│ B - seq: 2  (active) │
│ C - seq: 2  ⬆        │
│ D - seq: 3  ⬆        │
│ E - seq: 4  ⬆        │
│ F - seq: 6           │
└───────────────────────┘

Step 2: Move active to target
┌───────────────────────┐
│ A - seq: 1           │
│ C - seq: 2           │
│ D - seq: 3           │
│ E - seq: 4           │
│ B - seq: 5  ✓        │
│ F - seq: 6           │
└───────────────────────┘

Request Format​

POST /api/v4/core/PRD/sort
Content-Type: application/json
Authorization: Bearer {token}

{
  "active_id": "123",
  "active_sequence": 2,
  "over_id": "456",
  "over_sequence": 5,
  "sort_column_name": "XPRD40",
  "source": "productList"
}

Required Fields:

FieldTypeDescriptionExample
active_idstring/intID of item being moved"123"
active_sequenceintCurrent sequence number of active item2
over_idstring/intID of target position item"456"
over_sequenceintSequence number of target position5
sort_column_namestringField name containing sequence"XPRD40"
sourcestringSecurity area"productList"

Optional Fields:

FieldTypeDescriptionExample
extra_condstringSQL condition for scoped sorting"XPRD05 = 'electronics'"
skipOwnerbooleanSkip LOWNER updatefalse (default)
forceOwnerstringOverride LOWNER value"user123"
skipLDATAbooleanSkip LDATA updatefalse (default)
forceLDATAstringOverride LDATA value"20251219143000"

Response Format​

Success (200 OK):

{
  "code": 200,
  "message": "Sort operation completed successfully",
  "affectedRows": 4
}

Error (500):

{
  "code": 500,
  "msg": "Invalid body provided for Sort operation! Fix errors before retry.",
  "errors": [
    "Missing active_id",
    "Missing sort_column_name"
  ]
}

Sort Algorithms​

Scenario 1: Moving Down (active_sequence < over_sequence)​

Example: Drag item at position 2 to position 5

Algorithm:

  1. Shift items up between active and target positions:

    UPDATE TB_ANAG_{DIM}00
    SET {sort_field} = {sort_field} - 1
    WHERE {sort_field} <= {over_sequence}
      AND {sort_field} > {active_sequence}

  2. Move active item to target position:

    UPDATE TB_ANAG_{DIM}00
    SET {sort_field} = {over_sequence}
    WHERE {sort_field} = {active_sequence}
      AND {ID} = {active_id}

Visual:

Before:  A(1) B(2) C(3) D(4) E(5) F(6)
         ---- ↓              ↑
Step 1:  A(1) B(2) C(2) D(3) E(4) F(6)  [shift C,D,E up]
Step 2:  A(1) C(2) D(3) E(4) B(5) F(6)  [move B to 5]

Scenario 2: Moving Up (active_sequence > over_sequence)​

Example: Drag item at position 5 to position 2

Algorithm:

  1. Shift items down between target and active positions:

    UPDATE TB_ANAG_{DIM}00
    SET {sort_field} = {sort_field} + 1
    WHERE {sort_field} >= {over_sequence}
      AND {sort_field} < {active_sequence}

  2. Move active item to target position:

    UPDATE TB_ANAG_{DIM}00
    SET {sort_field} = {over_sequence}
    WHERE {sort_field} = {active_sequence}
      AND {ID} = {active_id}

Visual:

Before:  A(1) B(2) C(3) D(4) E(5) F(6)
              ↓              ↑ ----
Step 1:  A(1) B(3) C(4) D(5) E(5) F(6)  [shift B,C,D down]
Step 2:  A(1) E(2) B(3) C(4) D(5) F(6)  [move E to 2]

Transaction Safety​

ACID Guarantees:

BEGIN TRANSACTION
  UPDATE (shift items)
  UPDATE (move active item)
  UPDATE LOWNER, LDATA for all affected rows
COMMIT

Rollback Scenarios:

  • Database error during shift operation
  • Active item not found
  • Permission denied
  • Invalid sequence numbers

Conditional Sorting with extra_cond​

Use Case: Sort products within a specific category only

{
  "active_id": "123",
  "active_sequence": 2,
  "over_id": "456",
  "over_sequence": 5,
  "sort_column_name": "XPRD40",
  "extra_cond": "XPRD05 = 'electronics'",
  "source": "productList"
}

Effect:

  • Only products where XPRD05 = 'electronics' are affected
  • Products in other categories maintain their sequence
  • Enables grouped sorting (e.g., sort within parent, sort within status)

SQL Generated:

UPDATE TB_ANAG_PRD00
SET XPRD40 = XPRD40 - 1
WHERE XPRD40 <= 5
  AND XPRD40 > 2
  AND XPRD05 = 'electronics'

JavaScript Implementation​

Basic Drag-and-Drop Sort​

class SortableList {
  constructor(apiBase, token, dimension, sortField) {
    this.apiBase = apiBase;
    this.token = token;
    this.dimension = dimension;
    this.sortField = sortField;
  }

  async reorderItem(activeItem, overItem, extraCond = null) {
    const requestBody = {
      active_id: activeItem.id,
      active_sequence: activeItem.sequence,
      over_id: overItem.id,
      over_sequence: overItem.sequence,
      sort_column_name: this.sortField,
      source: 'productList'
    };

    if (extraCond) {
      requestBody.extra_cond = extraCond;
    }

    const response = await fetch(
      `${this.apiBase}/api/v4/core/${this.dimension}/sort`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.token}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify(requestBody)
      }
    );

    if (!response.ok) {
      const error = await response.json();
      throw new Error(error.msg || 'Sort operation failed');
    }

    return response.json();
  }
}

HTML Drag-and-Drop Integration​

class DraggableProductList {
  constructor(containerId, sorter) {
    this.container = document.getElementById(containerId);
    this.sorter = sorter;
    this.activeItem = null;
  }

  renderList(products) {
    this.container.innerHTML = '';

    products.forEach(product => {
      const item = document.createElement('div');
      item.className = 'product-item';
      item.draggable = true;
      item.dataset.id = product.PRD_ID;
      item.dataset.sequence = product.XPRD40;
      item.textContent = product.XPRD01;

      // Drag events
      item.addEventListener('dragstart', (e) => {
        this.activeItem = {
          id: item.dataset.id,
          sequence: parseInt(item.dataset.sequence)
        };
        item.classList.add('dragging');
      });

      item.addEventListener('dragend', (e) => {
        item.classList.remove('dragging');
      });

      item.addEventListener('dragover', (e) => {
        e.preventDefault(); // Allow drop
      });

      item.addEventListener('drop', async (e) => {
        e.preventDefault();

        const overItem = {
          id: item.dataset.id,
          sequence: parseInt(item.dataset.sequence)
        };

        if (this.activeItem.id === overItem.id) {
          return; // Same item
        }

        try {
          await this.sorter.reorderItem(this.activeItem, overItem);

          // Refresh list after successful sort
          await this.refreshList();
        } catch (error) {
          console.error('Sort failed:', error);
          alert(error.message);
        }
      });

      this.container.appendChild(item);
    });
  }

  async refreshList() {
    // Fetch updated list and re-render
    const response = await fetch(
      `/api/v4/core/PRD?source=productList&$order=XPRD40 ASC`,
      {
        headers: { 'Authorization': `Bearer ${token}` }
      }
    );

    const products = await response.json();
    this.renderList(products);
  }
}

// Usage
const sorter = new SortableList(apiBase, token, 'PRD', 'XPRD40');
const list = new DraggableProductList('product-list', sorter);

// Initial load
fetch('/api/v4/core/PRD?source=productList&$order=XPRD40 ASC', {
  headers: { 'Authorization': `Bearer ${token}` }
})
  .then(res => res.json())
  .then(products => list.renderList(products));

Best Practices​

✅ DO​

Always use transactions:

  • Sort operations are atomic
  • No manual transaction handling required
  • CoreWrite handles BEGIN/COMMIT automatically

Refresh list after sort:

await sorter.reorderItem(active, over);
await refreshProductList(); // Get updated sequences

Use extra_cond for grouped sorting:

// Sort only within same category
const extraCond = `XPRD05 = '${product.category}'`;
await sorter.reorderItem(active, over, extraCond);

Validate sequence numbers client-side:

if (active.sequence === over.sequence) {
  return; // No change needed
}

❌ DON'T​

Don't skip refreshing after sort:

// ❌ Wrong - UI may be out of sync
await sorter.reorderItem(active, over);
// (no refresh)

// ✅ Correct
await sorter.reorderItem(active, over);
await refreshList();

Don't forget source parameter:

// ❌ Wrong - permission validation fails
body: { active_id, active_sequence, over_id, over_sequence, sort_column_name }

// ✅ Correct
body: { ..., source: 'productList' }

Don't sort across different groups without extra_cond:

// ❌ Wrong - sorts all products globally
await sorter.reorderItem(active, over);

// ✅ Correct - sorts within category only
const extraCond = `XPRD05 = '${category}'`;
await sorter.reorderItem(active, over, extraCond);

Error Handling​

Missing Required Fields​

Request:

{
  "active_id": "123",
  "sort_column_name": "XPRD40"
}

Response (500):

{
  "code": 500,
  "msg": "Invalid body provided for Sort operation! Fix errors before retry.",
  "errors": [
    "Missing active_sequence",
    "Missing over_id",
    "Missing over_sequence"
  ]
}

Permission Denied​

Response (403):

{
  "code": 403,
  "message": "Insufficient permissions for sort operation",
  "required_grant": 2,
  "user_grant": 1
}

Use Case: Category-Scoped Product Sorting​

class CategoryProductSorter {
  constructor(apiBase, token) {
    this.apiBase = apiBase;
    this.token = token;
  }

  async loadCategory(categoryId) {
    const response = await fetch(
      `/api/v4/core/PRD?source=productList` +
      `&$filter=XPRD05 eq '${categoryId}'` +
      `&$order=XPRD40 ASC`,
      {
        headers: { 'Authorization': `Bearer ${this.token}` }
      }
    );

    return response.json();
  }

  async reorderInCategory(categoryId, activeItem, overItem) {
    const response = await fetch(
      `${this.apiBase}/api/v4/core/PRD/sort`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.token}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          active_id: activeItem.PRD_ID,
          active_sequence: activeItem.XPRD40,
          over_id: overItem.PRD_ID,
          over_sequence: overItem.XPRD40,
          sort_column_name: 'XPRD40',
          extra_cond: `XPRD05 = '${categoryId}'`,
          source: 'productList'
        })
      }
    );

    if (!response.ok) {
      throw new Error('Sort failed');
    }

    return response.json();
  }

  async handleDrop(categoryId, activeProduct, targetProduct) {
    try {
      await this.reorderInCategory(categoryId, activeProduct, targetProduct);

      // Refresh category products
      const updatedProducts = await this.loadCategory(categoryId);
      this.renderProducts(updatedProducts);
    } catch (error) {
      console.error('Sort error:', error);
      alert('Failed to reorder products');
    }
  }

  renderProducts(products) {
    // Render logic here
    console.log('Updated products:', products);
  }
}

Summary​

Sort operations provide:

  • ✅ Drag-and-drop reordering via POST endpoint
  • ✅ Automatic sequence recalculation with two algorithms
  • ✅ Transaction-safe updates (ACID)
  • ✅ Conditional sorting with extra_cond
  • ✅ Audit trail updates (LOWNER, LDATA)
  • ✅ Support for grouped sorting (category, parent, status)

Key Takeaways:

  1. Use POST /api/v4/core/{DIM}/sort for reordering
  2. Provide active and target item IDs with sequences
  3. Always specify sort_column_name (sequence field)
  4. Use extra_cond for grouped sorting
  5. Refresh list after successful sort
  6. Operations are transaction-safe and atomic
