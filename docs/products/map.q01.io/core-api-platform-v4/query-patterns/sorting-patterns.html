Sorting Patterns

Overview​

Sorting determines the order of query results using the $order parameter. Core APIs support single-field, multi-field, and mixed-direction sorting.

Single-Field Sort​

Ascending Order (ASC)​

GET /api/v4/core/PRD?source=productList&\
  $order=XPRD01 ASC

Result: Products sorted A→Z by name.

Descending Order (DESC)​

GET /api/v4/core/PRD?source=productList&\
  $order=XPRD02 DESC

Result: Products sorted highest→lowest price.

Default Direction​

ASC is default if omitted:

# These are equivalent:
$order=XPRD01
$order=XPRD01 ASC

Multi-Field Sort​

Two Fields​

Sort by category, then name:

GET /api/v4/core/PRD?source=productList&\
  $order=XPRD05 ASC,XPRD01 ASC

Result:

Electronics - Widget A
Electronics - Widget B
Toys - Ball
Toys - Car

Products grouped by category, alphabetically sorted within each category.

Three or More Fields​

Complex sorting:

GET /api/v4/core/PRD?source=productList&\
  $order=XPRD05 ASC,XPRD06 DESC,XPRD02 DESC

Priority:

  1. Category (ascending)
  2. Active status (descending - active first)
  3. Price (descending - highest first)

Mixed Sort Directions​

Ascending + Descending​

GET /api/v4/core/PRD?source=productList&\
  $order=XPRD05 ASC,XPRD02 DESC

Result: Categories A→Z, within each category products sorted by price (high→low).

Common Sorting Patterns​

Pattern 1: Alphabetical List​

$order=XPRD01 ASC

Use: Product lists, customer lists, any A-Z sorting.

Pattern 2: Most Recent First​

$order=CDATA DESC

Use: Activity feeds, recent orders, audit logs.

Pattern 3: Highest Value First​

$order=XPRD02 DESC

Use: Top products by price, highest revenue customers.

Pattern 4: Priority + Date​

$order=XPRD35 ASC,CDATA DESC

Use: Task lists (high priority first, then by date).

Pattern 5: Status + Name​

$order=XPRD06 DESC,XPRD01 ASC

Use: Active items first, then alphabetically.

Sorting by Indexed Fields​

Performance Impact​

Fast - indexed field:

# ✅ XPRD03 is counter (indexed)
$order=XPRD03 ASC

Slower - non-indexed field:

# ⚠️ XPRD04 is text (may not be indexed)
$order=XPRD04 ASC

Best Practice: Sort by indexed fields for tables with >10k records.

Null Handling​

Nulls in Sorting​

Ascending: Nulls typically appear first (database dependent). Descending: Nulls typically appear last.

$order=XPRD08 ASC
# Results: NULL, NULL, 1, 2, 3, ...

$order=XPRD08 DESC
# Results: ..., 3, 2, 1, NULL, NULL

JavaScript Examples​

Dynamic Sort Builder​

class SortBuilder {
  constructor() {
    this.sorts = [];
  }

  addSort(field, direction = 'ASC') {
    this.sorts.push(`${field} ${direction.toUpperCase()}`);
    return this;
  }

  build() {
    return this.sorts.join(',');
  }

  clear() {
    this.sorts = [];
    return this;
  }
}

// Usage
const sort = new SortBuilder()
  .addSort('XPRD05', 'ASC')
  .addSort('XPRD02', 'DESC')
  .build();

// Result: "XPRD05 ASC,XPRD02 DESC"

Sort State Management (React)​

function ProductTable() {
  const [sortField, setSortField] = useState('XPRD01');
  const [sortDirection, setSortDirection] = useState('ASC');

  function handleSort(field) {
    if (sortField === field) {
      // Toggle direction if same field
      setSortDirection(sortDirection === 'ASC' ? 'DESC' : 'ASC');
    } else {
      // New field, default ASC
      setSortField(field);
      setSortDirection('ASC');
    }
  }

  useEffect(() => {
    loadProducts({ $order: `${sortField} ${sortDirection}` });
  }, [sortField, sortDirection]);

  return (
    <table>
      <thead>
        <tr>
          <th onClick={() => handleSort('XPRD01')}>
            Name {sortField === 'XPRD01' && (sortDirection === 'ASC' ? '↑' : '↓')}
          </th>
          <th onClick={() => handleSort('XPRD02')}>
            Price {sortField === 'XPRD02' && (sortDirection === 'ASC' ? '↑' : '↓')}
          </th>
        </tr>
      </thead>
      {/* ... */}
    </table>
  );
}

Best Practices​

✅ DO:​

Sort by indexed fields:

$order=XPRD03 ASC  # Counter field (indexed)

Use ASC/DESC explicitly:

$order=XPRD01 ASC  # Clear intent

Combine with pagination:

$order=XPRD01 ASC&$num_rows=25&$offset=0

❌ DON'T:​

Don't sort large tables by text fields:

# ⚠️ Slow on 100k+ records
$order=XPRD04 ASC

Don't omit $order in pagination:

# ❌ Inconsistent page results
$num_rows=25&$offset=25  # Missing $order

Summary​

  • ✅ Use $order to control result order
  • ✅ Format: field ASC|DESC
  • ✅ Multiple fields: field1 ASC,field2 DESC
  • ✅ Sort by indexed fields for performance
  • ✅ Always include $order with pagination
