Error Handling

Overview​

CoreWrite provides structured error responses for all failure scenarios. Error responses include error code, message, field (if applicable), and additional context for debugging.

Error Categories:

  • Authentication Errors (401) - Token issues
  • Authorization Errors (403) - Permission denied
  • Validation Errors (400) - Field validation failures
  • Constraint Errors (400) - Database constraint violations
  • Not Found Errors (404) - Record doesn't exist
  • Conflict Errors (409) - Concurrent modification
  • Server Errors (500) - Unexpected failures

Error Response Structure​

Standard Error Format​

{
  "error": "ValidationError",
  "message": "Required field missing: XPRD01",
  "code": "REQUIRED_FIELD_MISSING",
  "status": 400,
  "field": "XPRD01",
  "fieldDescription": "Product name",
  "dimension": "PRD",
  "timestamp": "2025-12-19T16:00:00Z",
  "requestId": "req_550e8400"
}

Fields:

  • error: Error type (ValidationError, AuthenticationError, etc.)
  • message: Human-readable description
  • code: Machine-readable error code
  • status: HTTP status code
  • field: Field causing error (if applicable)
  • fieldDescription: User-friendly field name
  • dimension: Dimension involved
  • timestamp: When error occurred
  • requestId: Unique request identifier for tracing

Authentication Errors (401)​

TOKEN_INVALID​

Cause: JWT token signature invalid or malformed

Response:

{
  "error": "AuthenticationError",
  "message": "Invalid or expired token",
  "code": "TOKEN_INVALID",
  "status": 401
}

Client Action: Redirect to login, refresh token

TOKEN_EXPIRED​

Cause: JWT token past expiration time

Response:

{
  "error": "AuthenticationError",
  "message": "Token has expired",
  "code": "TOKEN_EXPIRED",
  "status": 401,
  "expiredAt": "2025-12-19T15:00:00Z"
}

Client Action: Refresh token or re-authenticate

TOKEN_MISSING​

Cause: No Authorization header provided

Response:

{
  "error": "AuthenticationError",
  "message": "Authorization header missing",
  "code": "TOKEN_MISSING",
  "status": 401
}

Client Action: Include Authorization: Bearer {token} header

Authorization Errors (403)​

GRANT_DENIED​

Cause: User lacks required permission for operation

Response:

{
  "error": "AuthorizationError",
  "message": "Insufficient permissions for this operation",
  "code": "GRANT_DENIED",
  "status": 403,
  "required_grant": "products.write",
  "user_grants": ["products.read"],
  "user": "user@example.com"
}

Client Action: Request additional permissions from admin, hide UI elements

Validation Errors (400)​

REQUIRED_FIELD_MISSING​

Cause: Required field not provided

Response:

{
  "error": "ValidationError",
  "message": "Required field missing: XPRD01",
  "code": "REQUIRED_FIELD_MISSING",
  "status": 400,
  "field": "XPRD01",
  "fieldDescription": "Product name",
  "dimension": "PRD"
}

Client Action: Highlight field, show error message

FIELD_NOT_CREATEABLE​

Cause: Field lacks 'N' flag in COD_ON_OFF (not allowed on POST)

Response:

{
  "error": "ValidationError",
  "message": "Field not allowed in create operation: XPRD03",
  "code": "FIELD_NOT_CREATEABLE",
  "status": 400,
  "field": "XPRD03",
  "cod_on_off": "LDR",
  "required_flag": "N",
  "dimension": "PRD"
}

Client Action: Remove field from request, it's auto-generated

FIELD_NOT_MODIFIABLE​

Cause: Field lacks 'M' flag in COD_ON_OFF (not allowed on PUT/PATCH)

Response:

{
  "error": "ValidationError",
  "message": "Field not modifiable: XPRD03",
  "code": "FIELD_NOT_MODIFIABLE",
  "status": 400,
  "field": "XPRD03",
  "cod_on_off": "LDR",
  "required_flag": "M",
  "dimension": "PRD"
}

Client Action: Remove field from request, it's read-only

INVALID_FIELD_VALUE​

Cause: Field value violates business rule

Response:

{
  "error": "ValidationError",
  "message": "Price must be greater than zero",
  "code": "INVALID_FIELD_VALUE",
  "status": 400,
  "field": "XPRD02",
  "value": -10.00,
  "constraint": "value > 0"
}

Client Action: Show validation error, correct value

Constraint Errors (400)​

FOREIGN_KEY_VIOLATION​

Cause: Referenced record doesn't exist

Response:

{
  "error": "ConstraintViolationError",
  "message": "Category not found: cat_invalid",
  "code": "FOREIGN_KEY_VIOLATION",
  "status": 400,
  "field": "XPRD05",
  "value": "cat_invalid",
  "referencedTable": "TB_ANAG_CAT00",
  "referencedField": "CAT_ID"
}

Client Action: Select valid category from dropdown

UNIQUE_CONSTRAINT_VIOLATION​

Cause: Duplicate value for unique field

Response:

{
  "error": "ConstraintViolationError",
  "message": "Product code already exists: PRD-2025-001",
  "code": "UNIQUE_CONSTRAINT_VIOLATION",
  "status": 400,
  "field": "XPRD03",
  "value": "PRD-2025-001",
  "constraint": "UNIQUE(XPRD03)"
}

Client Action: Generate new code or modify existing record

CHECK_CONSTRAINT_VIOLATION​

Cause: Value fails database CHECK constraint

Response:

{
  "error": "ConstraintViolationError",
  "message": "Check constraint violation: XPRD02 must be >= 0",
  "code": "CHECK_CONSTRAINT_VIOLATION",
  "status": 400,
  "field": "XPRD02",
  "value": -10.00,
  "constraint": "CHECK (XPRD02 >= 0)"
}

Client Action: Correct value to meet constraint

Not Found Errors (404)​

RECORD_NOT_FOUND​

Cause: Record with given ID doesn't exist

Response:

{
  "error": "NotFoundError",
  "message": "Product not found: 123",
  "code": "RECORD_NOT_FOUND",
  "status": 404,
  "dimension": "PRD",
  "recordId": "123"
}

Client Action: Show "record not found" message, redirect to list

DIMENSION_NOT_FOUND​

Cause: Dimension not registered in TB_DIM

Response:

{
  "error": "NotFoundError",
  "message": "Dimension not found: INVALID",
  "code": "DIMENSION_NOT_FOUND",
  "status": 404,
  "dimension": "INVALID"
}

Client Action: Check URL, use correct dimension name

Conflict Errors (409)​

CONCURRENT_MODIFICATION​

Cause: Record modified by another user (optimistic locking)

Response:

{
  "error": "ConflictError",
  "message": "Record was modified by another user",
  "code": "CONCURRENT_MODIFICATION",
  "status": 409,
  "dimension": "PRD",
  "recordId": "123",
  "expectedVersion": "20251219150000",
  "currentVersion": "20251219160000",
  "modifiedBy": "admin@example.com"
}

Client Action: Refresh record, show conflict resolution UI

Server Errors (500)​

INTERNAL_SERVER_ERROR​

Cause: Unexpected server error

Response:

{
  "error": "InternalServerError",
  "message": "An unexpected error occurred",
  "code": "INTERNAL_SERVER_ERROR",
  "status": 500,
  "requestId": "req_550e8400"
}

Client Action: Retry with exponential backoff, contact support

DATABASE_ERROR​

Cause: Database connection or query failure

Response:

{
  "error": "DatabaseError",
  "message": "Database operation failed",
  "code": "DATABASE_ERROR",
  "status": 500,
  "requestId": "req_550e8400"
}

Client Action: Retry, check system status page

Nested Errors​

Multiple Validation Errors​

Response with multiple field errors:

{
  "error": "ValidationError",
  "message": "Multiple validation errors",
  "code": "VALIDATION_ERRORS",
  "status": 400,
  "errors": [
    {
      "field": "XPRD01",
      "message": "Product name is required",
      "code": "REQUIRED_FIELD_MISSING"
    },
    {
      "field": "XPRD02",
      "message": "Price must be greater than zero",
      "code": "INVALID_FIELD_VALUE",
      "value": -10.00
    },
    {
      "field": "XPRD05",
      "message": "Category not found: cat_invalid",
      "code": "FOREIGN_KEY_VIOLATION",
      "value": "cat_invalid"
    }
  ]
}

JavaScript Error Handling​

Error Handler Class​

class CoreAPIError extends Error {
  constructor(response) {
    super(response.message);
    this.code = response.code;
    this.status = response.status;
    this.field = response.field;
    this.errors = response.errors;  // Nested errors
    this.requestId = response.requestId;
  }

  isAuthenticationError() {
    return this.status === 401;
  }

  isAuthorizationError() {
    return this.status === 403;
  }

  isValidationError() {
    return this.status === 400 && this.code.includes('FIELD');
  }

  isNotFoundError() {
    return this.status === 404;
  }

  isServerError() {
    return this.status >= 500;
  }
}

// Usage
async function createProduct(data) {
  try {
    const response = await fetch(`${apiBase}/api/v4/core/PRD`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ data })
    });

    if (!response.ok) {
      const errorData = await response.json();
      throw new CoreAPIError(errorData);
    }

    return response.json();
  } catch (error) {
    handleError(error);
  }
}

function handleError(error) {
  if (!(error instanceof CoreAPIError)) {
    console.error('Network error:', error);
    alert('Network error. Please check your connection.');
    return;
  }

  if (error.isAuthenticationError()) {
    // Redirect to login
    window.location.href = '/login';
    return;
  }

  if (error.isAuthorizationError()) {
    alert('Access denied. You don\'t have permission for this operation.');
    return;
  }

  if (error.isValidationError()) {
    showValidationErrors(error);
    return;
  }

  if (error.isNotFoundError()) {
    alert('Record not found.');
    window.location.href = '/products';
    return;
  }

  if (error.isServerError()) {
    alert('Server error. Please try again later.');
    return;
  }

  alert(`Error: ${error.message}`);
}

function showValidationErrors(error) {
  if (error.errors) {
    // Multiple errors
    error.errors.forEach(err => {
      highlightField(err.field, err.message);
    });
  } else {
    // Single error
    highlightField(error.field, error.message);
  }
}

function highlightField(fieldName, message) {
  const input = document.querySelector(`[name="${fieldName}"]`);
  if (input) {
    input.classList.add('error');
    showFieldError(input, message);
    input.focus();
  }
}

React Error Boundary​

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('Error caught by boundary:', error, errorInfo);

    // Log to error tracking service
    logErrorToService(error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="error-boundary">
          <h2>Something went wrong</h2>
          <p>{this.state.error?.message}</p>
          <button onClick={() => window.location.reload()}>
            Reload Page
          </button>
        </div>
      );
    }

    return this.props.children;
  }
}

Best Practices​

✅ DO:​

Handle specific error codes:

// ✅ Good - specific handling
if (error.code === 'REQUIRED_FIELD_MISSING') {
  highlightField(error.field);
} else if (error.code === 'FOREIGN_KEY_VIOLATION') {
  showRelatedRecordSelector(error.field);
}

Show user-friendly messages:

// ✅ Good - user-friendly
alert(`Please fill in ${error.fieldDescription}`);

Log errors for debugging:

// ✅ Good - logging
console.error('API Error:', {
  code: error.code,
  message: error.message,
  requestId: error.requestId
});

Retry on server errors:

// ✅ Good - retry with backoff
if (error.isServerError()) {
  await retryWithBackoff(() => createProduct(data));
}

❌ DON'T:​

Don't show technical codes to users:

// ❌ Bad - confusing
alert(error.code);  // "REQUIRED_FIELD_MISSING"

// ✅ Good - user-friendly
alert(`${error.fieldDescription} is required`);

Don't ignore errors:

// ❌ Bad - silent failure
try {
  await createProduct(data);
} catch (error) {
  // Ignore
}

Don't retry authentication errors:

// ❌ Bad - infinite retries
if (error.status === 401) {
  await retry(() => createProduct(data));
}

// ✅ Good - redirect to login
if (error.status === 401) {
  window.location.href = '/login';
}

Summary​

  • ✅ Structured error responses with code, message, field
  • ✅ HTTP status codes: 401 (auth), 403 (authz), 400 (validation), 404 (not found), 409 (conflict), 500 (server)
  • ✅ Machine-readable error codes for programmatic handling
  • ✅ Nested errors for multiple validation failures
  • ✅ Request ID for tracing and debugging
  • ✅ Field-specific errors with descriptions

Key Takeaways:

  1. Check error code for specific handling
  2. Show user-friendly messages (not technical codes)
  3. Highlight fields causing validation errors
  4. Redirect to login on 401 (authentication)
  5. Show access denied on 403 (authorization)
  6. Retry with backoff on 500 (server errors)
  7. Log request ID for debugging
