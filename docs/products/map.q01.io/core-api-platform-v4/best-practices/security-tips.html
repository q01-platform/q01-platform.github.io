Security Tips

Overview​

Practical security tips for hardening Q01 Core API integrations against common threats. Security is multi-layered: authentication, authorization, input validation, data protection, and monitoring.

Security Layers:

  1. Authentication (JWT tokens)
  2. Authorization (TB_MENU grants)
  3. Input Validation
  4. Data Protection
  5. Rate Limiting
  6. Audit Trail
  7. Monitoring & Alerts

Tip 1: Secure Token Storage​

Problem​

Tokens exposed in localStorage or code are vulnerable to XSS attacks.

Solution​

// ❌ INSECURE - localStorage (XSS vulnerable!)
localStorage.setItem('accessToken', token);

// ❌ INSECURE - Hardcoded
const TOKEN = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

// ✅ SECURE - httpOnly cookie (server-side)
res.cookie('accessToken', token, {
  httpOnly: true,        // Not accessible via JavaScript
  secure: true,          // HTTPS only
  sameSite: 'strict',    // CSRF protection
  maxAge: 900000,        // 15 minutes
  domain: '.q01.io',
  path: '/'
});

// ✅ SECURE - Environment variable (server-side)
const TOKEN = process.env.CORE_API_TOKEN;

// ✅ ACCEPTABLE - sessionStorage (better than localStorage)
sessionStorage.setItem('accessToken', token);  // Cleared on tab close

Best practices:

  • Server-side: httpOnly cookies or environment variables
  • Client-side: sessionStorage only (never localStorage)
  • Never hardcode tokens in code
  • Use short expiration times (15 minutes for access tokens)

Impact: Prevents token theft via XSS

Tip 2: Implement Token Refresh​

Problem​

Long-lived tokens increase security risk.

Solution​

class SecureAPIClient {
  constructor() {
    this.accessToken = null;
    this.refreshToken = null;
    this.tokenExpiry = null;
  }

  async request(url, options) {
    // Check if token expired
    if (this.isTokenExpired()) {
      await this.refreshAccessToken();
    }

    return fetch(url, {
      ...options,
      headers: {
        ...options.headers,
        'Authorization': `Bearer ${this.accessToken}`
      }
    });
  }

  isTokenExpired() {
    if (!this.tokenExpiry) return true;
    return Date.now() >= this.tokenExpiry - 60000;  // Refresh 1 min before
  }

  async refreshAccessToken() {
    try {
      const response = await fetch('https://api.q01.io/api/v4/auth/refresh', {
        method: 'POST',
        headers: {'Content-Type': 'application/json'},
        body: JSON.stringify({refreshToken: this.refreshToken})
      });

      const {accessToken, expiresIn} = await response.json();

      this.accessToken = accessToken;
      this.tokenExpiry = Date.now() + (expiresIn * 1000);
    } catch (error) {
      // Refresh failed - redirect to login
      window.location.href = '/login';
    }
  }
}

Token lifetimes:

  • Access token: 15 minutes
  • Refresh token: 7 days
  • Auto-refresh: 1 minute before expiry

Impact: Limits token exposure window

Tip 3: Validate All Input​

Problem​

Unvalidated input enables SQL injection, XSS, and data corruption.

Solution​

const validator = require('validator');

function validateProductInput(data) {
  const errors = [];

  // Required fields
  if (!data.XPRD01 || validator.isEmpty(data.XPRD01)) {
    errors.push('Product name is required');
  }

  // Type validation
  if (typeof data.XPRD02 !== 'number' || data.XPRD02 <= 0) {
    errors.push('Price must be a positive number');
  }

  // Length validation
  if (data.XPRD01 && data.XPRD01.length > 255) {
    errors.push('Product name too long (max 255 characters)');
  }

  // Format validation
  if (data.XPRD03 && !validator.isAlphanumeric(data.XPRD03.replace(/-/g, ''))) {
    errors.push('Product code must be alphanumeric');
  }

  // Sanitize HTML
  if (data.XPRD01) {
    data.XPRD01 = validator.escape(data.XPRD01);
  }

  if (errors.length > 0) {
    throw new ValidationError(errors);
  }

  return data;
}

// Usage
app.post('/api/products', async (req, res) => {
  try {
    const validatedData = validateProductInput(req.body);
    const product = await coreAPI.post('/api/v4/core/products', validatedData);
    res.json(product);
  } catch (error) {
    if (error instanceof ValidationError) {
      return res.status(400).json({errors: error.errors});
    }
    throw error;
  }
});

Validation checklist:

  • Required fields present
  • Correct data types
  • Length constraints
  • Format validation (email, phone, etc.)
  • Range validation (min/max)
  • Whitelist allowed characters
  • Sanitize HTML
  • Escape special characters

Impact: Prevents injection attacks and data corruption

Tip 4: Always Check Grants​

Problem​

Performing operations without permission checks.

Solution​

class GrantChecker {
  constructor(coreAPI) {
    this.coreAPI = coreAPI;
    this.grants = null;
  }

  async loadGrants() {
    const response = await this.coreAPI.get('/api/v4/core/menu');
    this.grants = response.data;
  }

  hasGrant(dimension, operation) {
    if (!this.grants) {
      throw new Error('Grants not loaded');
    }

    const menu = this.grants.find(m => m.MENU_CODE === dimension);

    if (!menu) return false;

    // Check operation grant
    const grantMap = {
      'read': 'R',
      'create': 'C',
      'update': 'U',
      'delete': 'D'
    };

    const requiredGrant = grantMap[operation];
    return menu.GRANTS.includes(requiredGrant);
  }

  requireGrant(dimension, operation) {
    if (!this.hasGrant(dimension, operation)) {
      throw new Error(`Permission denied: ${operation} grant required for ${dimension}`);
    }
  }
}

// Usage
const grantChecker = new GrantChecker(coreAPI);
await grantChecker.loadGrants();

// Before operations
async function deleteProduct(id) {
  // Check grant first
  grantChecker.requireGrant('PRD', 'delete');

  // Proceed with delete
  return coreAPI.delete(`/api/v4/core/products/${id}`);
}

Grant checking strategy:

  • Check grants on page load
  • Cache grants in memory
  • Re-check before sensitive operations
  • Show/hide UI based on grants

Impact: Prevents unauthorized operations

Tip 5: Enforce Rate Limiting​

Problem​

APIs vulnerable to abuse without rate limiting.

Solution​

const rateLimit = require('express-rate-limit');
const RedisStore = require('rate-limit-redis');
const Redis = require('ioredis');

// Create Redis client
const redis = new Redis({
  host: 'redis',
  port: 6379
});

// Configure rate limiter
const limiter = rateLimit({
  store: new RedisStore({
    client: redis,
    prefix: 'ratelimit:'
  }),
  windowMs: 15 * 60 * 1000,  // 15 minutes
  max: 100,                   // Max 100 requests per window
  standardHeaders: true,      // Return rate limit info in headers
  legacyHeaders: false,
  handler: (req, res) => {
    res.status(429).json({
      error: 'Too many requests',
      retryAfter: req.rateLimit.resetTime
    });
  },
  skip: (req) => {
    // Skip rate limiting for internal services
    return req.ip === '10.0.0.1';
  }
});

// Apply to all routes
app.use('/api/', limiter);

// Stricter limit for sensitive operations
const strictLimiter = rateLimit({
  windowMs: 60 * 60 * 1000,  // 1 hour
  max: 10                     // Max 10 requests per hour
});

app.post('/api/auth/login', strictLimiter);
app.post('/api/auth/register', strictLimiter);

Rate limit tiers:

  • Anonymous: 10 req/min
  • Authenticated: 100 req/min
  • Premium: 1000 req/min
  • Internal: Unlimited

Impact: Prevents abuse and DDoS attacks

Tip 6: Implement CORS Properly​

Problem​

Incorrect CORS allows unauthorized access.

Solution​

const cors = require('cors');

// ❌ INSECURE - Allow all origins
app.use(cors({origin: '*'}));

// ✅ SECURE - Whitelist specific origins
const allowedOrigins = [
  'https://app.q01.io',
  'https://admin.q01.io',
  'https://mobile.q01.io'
];

app.use(cors({
  origin: (origin, callback) => {
    // Allow requests with no origin (mobile apps, Postman, etc.)
    if (!origin) return callback(null, true);

    if (allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true,  // Allow cookies
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Source', 'X-Peso'],
  exposedHeaders: ['X-RateLimit-Remaining', 'X-RateLimit-Reset'],
  maxAge: 86400  // Cache preflight for 24h
}));

CORS best practices:

  • Whitelist specific origins (never use * in production)
  • Enable credentials only when needed
  • Restrict allowed methods
  • Validate origin on server-side
  • Set appropriate maxAge for preflight caching

Impact: Prevents unauthorized cross-origin access

Tip 7: Enable HTTPS Only​

Problem​

Unencrypted HTTP exposes data in transit.

Solution​

// Redirect HTTP to HTTPS
app.use((req, res, next) => {
  if (req.protocol === 'http') {
    return res.redirect(301, `https://${req.headers.host}${req.url}`);
  }
  next();
});

// Force HTTPS (Strict-Transport-Security)
const helmet = require('helmet');

app.use(helmet({
  hsts: {
    maxAge: 31536000,        // 1 year
    includeSubDomains: true,
    preload: true
  },
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'"],
      scriptSrc: ["'self'"],
      imgSrc: ["'self'", 'data:', 'https:'],
      connectSrc: ["'self'", 'https://api.q01.io']
    }
  },
  referrerPolicy: {policy: 'strict-origin-when-cross-origin'}
}));

SSL/TLS configuration:

  • Use TLS 1.2 or higher
  • Strong cipher suites only
  • Valid certificate from trusted CA
  • Enable HTTP/2 for better performance

Impact: Protects data in transit

Tip 8: Log Security Events​

Problem​

Security incidents go undetected without logging.

Solution​

const winston = require('winston');

// Create security logger
const securityLogger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({filename: 'security.log'})
  ]
});

// Log security events
function logSecurityEvent(event, details) {
  securityLogger.info({
    timestamp: new Date().toISOString(),
    event: event,
    ...details
  });
}

// Middleware to log authentication
app.use('/api/', (req, res, next) => {
  logSecurityEvent('api_request', {
    method: req.method,
    url: req.url,
    ip: req.ip,
    userAgent: req.headers['user-agent'],
    userId: req.user?.id
  });
  next();
});

// Log failed authentication
app.post('/api/auth/login', async (req, res) => {
  try {
    const user = await authenticate(req.body);
    res.json({token: user.token});
  } catch (error) {
    logSecurityEvent('failed_login', {
      username: req.body.username,
      ip: req.ip,
      reason: error.message
    });
    res.status(401).json({error: 'Invalid credentials'});
  }
});

// Alert on suspicious activity
function detectSuspiciousActivity(userId, ip) {
  // Check for unusual patterns
  const recentAttempts = getRecentLoginAttempts(userId, ip);

  if (recentAttempts > 5) {
    logSecurityEvent('suspicious_activity', {
      type: 'multiple_failed_logins',
      userId: userId,
      ip: ip,
      count: recentAttempts
    });

    // Send alert
    sendSecurityAlert({
      title: 'Suspicious Login Activity',
      userId: userId,
      ip: ip
    });
  }
}

Events to log:

  • Authentication attempts (success/failure)
  • Authorization failures
  • Data access (sensitive records)
  • Data modifications (create/update/delete)
  • Configuration changes
  • API errors
  • Rate limit violations

Impact: Enables detection and response to security incidents

Tip 9: Implement CSRF Protection​

Problem​

Cross-Site Request Forgery attacks.

Solution​

const csrf = require('csurf');
const cookieParser = require('cookie-parser');

// Configure CSRF protection
app.use(cookieParser());

const csrfProtection = csrf({
  cookie: {
    httpOnly: true,
    secure: true,
    sameSite: 'strict'
  }
});

// Apply to state-changing operations
app.post('/api/products', csrfProtection, async (req, res) => {
  // CSRF token validated automatically
  const product = await coreAPI.post('/api/v4/core/products', req.body);
  res.json(product);
});

// Provide token to frontend
app.get('/api/csrf-token', csrfProtection, (req, res) => {
  res.json({csrfToken: req.csrfToken()});
});

// Frontend - Include token in requests
const csrfResponse = await fetch('/api/csrf-token');
const {csrfToken} = await csrfResponse.json();

await fetch('/api/products', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRF-Token': csrfToken
  },
  body: JSON.stringify(productData)
});

CSRF protection strategies:

  • Synchronizer token pattern (CSRF tokens)
  • Double-submit cookie
  • SameSite cookie attribute
  • Custom request headers

Impact: Prevents unauthorized state-changing requests

Tip 10: Monitor and Alert​

Problem​

Security incidents detected too late.

Solution​

class SecurityMonitor {
  constructor() {
    this.alerts = [];
  }

  checkFailedLogins(userId) {
    const recentFailures = this.getRecentFailures(userId);

    if (recentFailures > 5) {
      this.sendAlert({
        level: 'warning',
        title: 'Multiple Failed Logins',
        userId: userId,
        count: recentFailures
      });
    }

    if (recentFailures > 10) {
      // Lock account
      this.lockAccount(userId);

      this.sendAlert({
        level: 'critical',
        title: 'Account Locked',
        userId: userId,
        reason: 'Too many failed login attempts'
      });
    }
  }

  checkUnusualActivity(userId, action) {
    const pattern = this.getUserPattern(userId);

    // Check for unusual time
    if (this.isUnusualTime(pattern)) {
      this.sendAlert({
        level: 'info',
        title: 'Unusual Access Time',
        userId: userId,
        time: new Date().toISOString()
      });
    }

    // Check for unusual location
    if (this.isUnusualLocation(pattern)) {
      this.sendAlert({
        level: 'warning',
        title: 'Unusual Access Location',
        userId: userId,
        location: pattern.location
      });
    }

    // Check for privilege escalation
    if (this.isPrivilegeEscalation(action)) {
      this.sendAlert({
        level: 'critical',
        title: 'Privilege Escalation Attempt',
        userId: userId,
        action: action
      });
    }
  }

  async sendAlert(alert) {
    console.error('[SECURITY ALERT]', alert);

    // Send to monitoring service
    await fetch(process.env.ALERT_WEBHOOK_URL, {
      method: 'POST',
      headers: {'Content-Type': 'application/json'},
      body: JSON.stringify(alert)
    });

    // Store alert
    this.alerts.push({
      ...alert,
      timestamp: Date.now()
    });
  }
}

Monitoring checklist:

  • Failed authentication attempts
  • Unusual access patterns
  • Privilege escalation attempts
  • Data exfiltration attempts
  • Rate limit violations
  • API errors
  • Configuration changes

Impact: Early detection and response to threats

Security Checklist​

Authentication​

  • Use JWT tokens
  • Store tokens securely (httpOnly cookies)
  • Implement token refresh
  • Short token lifetimes (15 min access, 7 days refresh)
  • Revoke tokens on logout

Authorization​

  • Check TB_MENU grants before operations
  • Implement role-based access control
  • Validate permissions server-side
  • Cache grants efficiently
  • Log authorization failures

Input Validation​

  • Validate all user input
  • Sanitize HTML content
  • Escape special characters
  • Enforce type constraints
  • Check length limits

Data Protection​

  • Use HTTPS only
  • Enable HSTS
  • Implement CORS properly
  • Encrypt sensitive data at rest
  • Mask sensitive data in logs

Infrastructure​

  • Implement rate limiting
  • Enable CSRF protection
  • Configure security headers (helmet)
  • Keep dependencies updated
  • Regular security audits

Monitoring​

  • Log security events
  • Set up alerts
  • Monitor failed logins
  • Track unusual activity
  • Regular security reviews

Summary​

Top 5 Security Must-Haves:

  1. ✅ Secure token storage (httpOnly cookies)
  2. ✅ Input validation (prevent injection)
  3. ✅ Grant checking (verify permissions)
  4. ✅ Rate limiting (prevent abuse)
  5. ✅ Security monitoring (detect threats)

Key Takeaways:

  • Security is multi-layered (defense in depth)
  • Validate everything (never trust user input)
  • Log security events (enable detection)
  • Monitor continuously (proactive defense)
  • Keep dependencies updated (patch vulnerabilities)

Security = Authentication + Authorization + Validation + Monitoring
