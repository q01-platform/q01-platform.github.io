Performance Tips

Overview​

Quick reference for optimizing Q01 Core API performance. These tips provide immediate improvements with minimal effort.

Quick Wins:

  1. Select only required fields
  2. Use keyset pagination
  3. Implement caching
  4. Batch requests
  5. Use connection pooling
  6. Optimize queries with indexes
  7. Compress responses
  8. Monitor and profile

Tip 1: Select Only Required Fields​

Problem​

Fetching all fields wastes bandwidth and processing time.

Solution​

// ❌ Slow - Fetches all 50 fields
const products = await api.get('/api/v4/core/products?limit=100');

// ✅ Fast - Fetches only 5 fields (10x faster!)
const products = await api.get('/api/v4/core/products', {
  params: {
    limit: 100,
    fields: 'PRD_ID,XPRD01,XPRD02,XPRD03,XPRD05'
  }
});

Impact: 5-10x faster response, 90% less bandwidth

Tip 2: Use Keyset Pagination​

Problem​

OFFSET pagination slows down with large datasets.

Solution​

// ❌ Slow - OFFSET pagination (scans 1M rows!)
const page50 = await api.get('/api/v4/core/products', {
  params: {limit: 100, offset: 1000000}
});
// ~10 seconds!

// ✅ Fast - Keyset pagination (constant time)
const page50 = await api.get('/api/v4/core/products', {
  params: {
    limit: 100,
    order_by: 'PRD_ID',
    after: lastId  // ID from previous page
  }
});
// ~50ms (always!)

Impact: 100-1000x faster for deep pagination

Tip 3: Implement Client-Side Caching​

Problem​

Repeated requests for same data waste resources.

Solution​

class CachedClient {
  constructor(ttl = 300000) {
    this.cache = new Map();
    this.ttl = ttl;
  }

  async get(url) {
    const cached = this.cache.get(url);

    if (cached && Date.now() - cached.ts < this.ttl) {
      return cached.data;  // Cache hit!
    }

    const data = await api.get(url);
    this.cache.set(url, {data, ts: Date.now()});

    return data;
  }

  invalidate(url) {
    this.cache.delete(url);
  }
}

// Usage
const client = new CachedClient(300000);  // 5 min TTL

// First call: API request
await client.get('/api/v4/core/products/123');

// Subsequent calls: Cache hit (instant!)
await client.get('/api/v4/core/products/123');
await client.get('/api/v4/core/products/123');

Impact: 100x faster for repeated requests, reduced server load

Tip 4: Batch Requests​

Problem​

Making many individual requests has high overhead.

Solution​

// ❌ Slow - 100 separate requests
for (const product of products) {
  await api.patch(`/api/v4/core/products/${product.id}`, {
    XPRD02: product.newPrice
  });
}
// ~10 seconds

// ✅ Fast - Single batch request
await api.put('/api/v4/core/products/batch', {
  records: products.map(p => ({
    PRD_ID: p.id,
    XPRD02: p.newPrice
  }))
});
// ~500ms

Impact: 10-20x faster for bulk operations

Tip 5: Use Connection Pooling​

Problem​

Creating new connections is expensive.

Solution​

// ❌ Slow - New connection per request
const mysql = require('mysql');
function query(sql) {
  const conn = mysql.createConnection({...});
  return conn.query(sql);
}

// ✅ Fast - Connection pool
const pool = mysql.createPool({
  host: 'db',
  user: 'user',
  password: 'pass',
  connectionLimit: 10,
  queueLimit: 0
});

function query(sql) {
  return pool.query(sql);
}

Configuration:

  • Small app: 5-10 connections
  • Medium app: 10-20 connections
  • Large app: 20-50 connections

Impact: 2-5x faster response times

Tip 6: Add Database Indexes​

Problem​

Queries scan entire tables without indexes.

Solution​

-- ❌ Slow - Full table scan
EXPLAIN SELECT * FROM TB_ANAG_PRD00 WHERE XPRD05 = 5;
-- type: ALL, rows: 1000000 (slow!)

-- ✅ Fast - Add index
CREATE INDEX idx_prd_category ON TB_ANAG_PRD00(XPRD05);

-- Now much faster!
EXPLAIN SELECT * FROM TB_ANAG_PRD00 WHERE XPRD05 = 5;
-- type: ref, rows: 1000 (fast!)

Common indexes to add:

-- Category filter
CREATE INDEX idx_prd_category ON TB_ANAG_PRD00(XPRD05);

-- Active products
CREATE INDEX idx_prd_active ON TB_ANAG_PRD00(XPRD06);

-- Composite for common filters
CREATE INDEX idx_prd_cat_active ON TB_ANAG_PRD00(XPRD05, XPRD06);

-- Date range queries
CREATE INDEX idx_ord_date ON TB_ANAG_ORD00(XORD_DATE);

Impact: 10-1000x faster queries

Tip 7: Enable Response Compression​

Problem​

Large JSON payloads increase transfer time.

Solution​

// Client - Request gzip compression
const axios = require('axios');
const api = axios.create({
  baseURL: 'https://api.q01.io',
  headers: {
    'Accept-Encoding': 'gzip, deflate'
  }
});

// Server - Enable compression (Express)
const compression = require('compression');
app.use(compression({
  level: 6,          // Compression level (1-9)
  threshold: 1024    // Only compress > 1KB
}));

Impact: 60-80% smaller payloads, 2-3x faster transfer

Tip 8: Prevent N+1 Queries​

Problem​

Loading related data in loops.

Solution​

// ❌ Slow - N+1 queries (101 requests!)
const products = await api.get('/api/v4/core/products?limit=100');

for (const product of products.data) {
  product.category = await api.get(
    `/api/v4/core/categories/${product.XPRD05}`
  );
}

// ✅ Fast - Use expand (1 request!)
const products = await api.get('/api/v4/core/products', {
  params: {
    limit: 100,
    expand: 'XPRD05'  // Auto-loads categories
  }
});

// ✅ Alternative - Batch load (2 requests)
const products = await api.get('/api/v4/core/products?limit=100');

const categoryIds = [...new Set(products.data.map(p => p.XPRD05))];
const categories = await api.get('/api/v4/core/categories', {
  params: {filters: `CAT_ID:in:${categoryIds.join(',')}`}
});

// Map in memory
const catMap = Object.fromEntries(categories.data.map(c => [c.CAT_ID, c]));
products.data.forEach(p => p.category = catMap[p.XPRD05]);

Impact: 50-100x faster (101 requests → 1-2 requests)

Tip 9: Use Appropriate Limits​

Problem​

Fetching too many records at once.

Solution​

// ❌ Slow - Too many records
const products = await api.get('/api/v4/core/products?limit=10000');
// High memory, slow processing

// ✅ Fast - Reasonable limit
const products = await api.get('/api/v4/core/products?limit=50');
// Fast, manageable

// ✅ Better - Paginate with generator
async function* fetchProducts(batchSize = 100) {
  let offset = 0;
  let hasMore = true;

  while (hasMore) {
    const batch = await api.get('/api/v4/core/products', {
      params: {limit: batchSize, offset}
    });

    yield batch.data;

    hasMore = batch.data.length === batchSize;
    offset += batchSize;
  }
}

// Process in batches
for await (const batch of fetchProducts(100)) {
  processBatch(batch);
}

Recommended limits:

  • List views: 20-50
  • Dropdowns: 100-200
  • Batch processing: 100-500

Impact: 5-10x faster initial load

Tip 10: Parallel Requests​

Problem​

Sequential requests waste time.

Solution​

// ❌ Slow - Sequential (3 seconds)
const products = await api.get('/api/v4/core/products');    // 1s
const categories = await api.get('/api/v4/core/categories'); // 1s
const customers = await api.get('/api/v4/core/customers');   // 1s

// ✅ Fast - Parallel (1 second!)
const [products, categories, customers] = await Promise.all([
  api.get('/api/v4/core/products'),
  api.get('/api/v4/core/categories'),
  api.get('/api/v4/core/customers')
]);

Impact: 2-3x faster for independent requests

Tip 11: Monitor Slow Queries​

Problem​

Not knowing which queries are slow.

Solution​

// Add timing middleware
api.interceptors.request.use(config => {
  config.metadata = {startTime: Date.now()};
  return config;
});

api.interceptors.response.use(response => {
  const duration = Date.now() - response.config.metadata.startTime;

  // Log slow queries
  if (duration > 1000) {
    console.warn(`Slow request: ${response.config.url} (${duration}ms)`);
  }

  return response;
});

Set alerts:

  • Warning: > 1s
  • Critical: > 3s

Impact: Identify and fix bottlenecks proactively

Tip 12: Use HTTP/2​

Problem​

HTTP/1.1 has connection limits.

Solution​

// Server - Enable HTTP/2
const http2 = require('http2');
const server = http2.createSecureServer({
  key: fs.readFileSync('key.pem'),
  cert: fs.readFileSync('cert.pem')
});

// Client - Use HTTP/2
const axios = require('axios');
const http2Adapter = require('axios-http2-adapter');

const api = axios.create({
  baseURL: 'https://api.q01.io',
  adapter: http2Adapter
});

Benefits:

  • Multiplexing (many requests over single connection)
  • Header compression
  • Server push

Impact: 20-30% faster for multiple requests

Quick Checklist​

Before Optimization​

  • Profile queries (identify slow ones)
  • Check database indexes
  • Monitor cache hit rate
  • Review connection pool settings
  • Analyze network traffic

During Optimization​

  • Add missing indexes
  • Implement caching layer
  • Optimize query patterns
  • Use batch operations
  • Enable compression

After Optimization​

  • Measure improvement
  • Set up monitoring
  • Configure alerts
  • Document changes
  • Share best practices

Performance Targets​

API Response Times (95th percentile)​

  • Single record GET: < 50ms
  • List (10 records): < 100ms
  • List (100 records): < 200ms
  • Search with filters: < 300ms
  • Single POST/PUT: < 100ms
  • Batch (100 records): < 1000ms

Database Queries​

  • Simple SELECT: < 10ms
  • JOIN query: < 50ms
  • Aggregation: < 100ms
  • Complex query: < 500ms

Caching​

  • Cache hit rate: > 80%
  • Cache TTL: 5-15 minutes
  • Cache miss overhead: < 10ms

Monitoring Tools​

Application Performance​

// Prometheus metrics
const promClient = require('prom-client');

const httpDuration = new promClient.Histogram({
  name: 'http_request_duration_ms',
  help: 'HTTP request duration',
  labelNames: ['method', 'route', 'status']
});

const cacheHits = new promClient.Counter({
  name: 'cache_hits_total',
  help: 'Cache hits',
  labelNames: ['cache']
});

Database Performance​

-- MySQL slow query log
SET GLOBAL slow_query_log = 1;
SET GLOBAL long_query_time = 1;

-- Check slow queries
SELECT * FROM mysql.slow_log
ORDER BY start_time DESC
LIMIT 10;

Network Performance​

# Test API latency
curl -w "@curl-format.txt" -o /dev/null -s https://api.q01.io/api/v4/core/products

# curl-format.txt:
# time_namelookup:  %{time_namelookup}s\n
